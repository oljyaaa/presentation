✦ 01 · ПРЕДМЕТНА ОБЛАСТЬ

Косметологічний бізнес
у цифрову еру

Салон краси — це складна сервісна система, де критично важливо координувати майстрів, відстежувати клієнтів та миттєво реагувати на нові запити. Beauty Room Workspace App автоматизує ці процеси.

💅
Послуги салону
Масаж, Elos-епіляція, Доглядові процедури — три ключові напрямки, кожен з яких обслуговує окремий майстер із власним розкладом і клієнтурою.
📅
Записи та розклад
Кожен запис містить клієнта, послугу, майстра, дату й час. Система дозволяє вести як активні записи, так і архів виконаних послуг.
📞
Клієнтські заявки
З вебсайту салону надходять запити на зворотній дзвінок — адміністратор бачить їх миттєво через push-сповіщення і може одразу записати клієнта.
👥
Ролева система
Creator / Admin — повний доступ до всіх даних.
Майстер — бачить лише свою категорію записів (Масаж, Elos, Догляд).
💰
Прайс-лист
Адміністратор може переглядати, додавати та редагувати послуги та їхні ціни прямо в мобільному застосунку, без відкриття вебпанелі.
📊
Звітність
Генерація PDF-звіту по активних записах із можливістю одразу поділитись файлом через будь-який месенджер або зберегти локально.

Контекст салону: Косметологічний салон Beeauty Room — сучасний заклад краси у місті Бердичів. Власниця — Островська Ольга Василівна — є безпосереднім замовником системи. Салон має декілька спеціалістів різного профілю, активно залучає клієнтів через власний сайт та потребує зручного інструменту для щоденного управління потоком записів.

✦ 02 · ДОЦІЛЬНІСТЬ

Навіщо мобільний застосунок?

Порівняльний аналіз підходів до управління косметологічним салоном.

❌ До впровадження

  • Записи велись у паперовому журналі або в Excel
  • Адміністратор не отримував сповіщень про нові запити з сайту
  • Майстри не мали доступу до свого розкладу поза робочим місцем
  • Прайс-лист оновлювався вручну на сайті (технічні знання)
  • Генерація звітів — ручна, займала значний час
  • Відсутність архівації виконаних записів

✅ Після впровадження

  • Всі записи — в єдиній системі, доступній з телефону 24/7
  • Миттєві push-сповіщення про нові заявки з сайту (опитування кожні 15 с)
  • Кожен майстер бачить тільки свої записи — конфіденційність
  • Прайс редагується прямо з телефону без технічних знань
  • PDF-звіт генерується натисканням однієї кнопки
  • Автоматичне архівування + відновлення записів

Ключова перевага: Застосунок інтегрується з вже наявним вебсайтом салону — дані з форм запису клієнтів автоматично потрапляють до веб-адміністративної панелі, звідки мобільний застосунок отримує їх через REST API. Жодних дублювань, єдина точка правди.

✦ 06 · API ІНТЕГРАЦІЯ

Єдиний REST API ендпоінт

Всі запити спрямовуються на https://thebeauty-room.com/api.php. Тип операції визначається параметром action.

GET-запити (отримання даних)

GET ?action=get_all_data Всі записи, архів, дзвінки
GET ?action=get_pricing Прайс-лист категоріями
GET ?action=get_profile&user_id=… Дані профілю майстра

POST: Авторизація та профіль

POST action=login Вхід (email, password)
POST action=update_profile Оновлення імені, тел., спец.
POST action=upload_avatar Завантаження фото профілю

POST: Записи клієнтів

POST action=add_record Новий запис
POST action=update_record Редагування запису
POST action=delete_record Видалення активного
POST action=move_to_archive Архівація запису
POST action=restore Відновлення з архіву
POST action=delete_archive Видалення з архіву
POST action=delete_appointment Видалення дзвінка

POST: Прайс та сповіщення

POST action=add_service Нова послуга
POST action=update_service Редагування послуги
POST action=check_new_appointments Перевірка нових заявок
POST action=add_callback Тест push-сповіщень
// Приклад: polling нових заявок (index.tsx) const formData = new FormData(); formData.append('action', 'check_new_appointments'); formData.append('last_check_time', lastCheckTime.current); const response = await fetch('https://thebeauty-room.com/api.php', { method: 'POST', body: formData }); const json = await response.json(); // { status, new_items[], server_time }
✦ 07 · ТЕСТУВАННЯ

Методи та тест-кейси

🧪 Методи тестування

✅ Модульне (Unit) тестування
Jest + @testing-library/react-native. Перевірка ізольованих функцій: маппінг API-даних, фільтрація записів, форматування дати/часу, логіка доступу canAccess().
🔗 Інтеграційне тестування
businessLogic.test.ts — тестування взаємодії між модулями AppConfig, Data, компонентами. Перевірка маппінгу record/archive/call в єдиний формат.
🎭 Компонентне тестування
Рендеринг LoginScreen, RecordCard, RecordFormModal, CategoryModal, StandardDashboard. Перевірка UI-станів та обробників подій через fireEvent.
◑ Тестування методом «сірої скриньки»
Тестувальник знає внутрішню логіку (ролі, permissions, API actions), але взаємодіє через реальний UI. Перевіряються граничні стани: порожні поля, мережеві помилки, некоректна відповідь сервера.

◑ Тестування методом «Сірої скриньки» (Grey-box Testing)

Сіра скринька дозволила тестувальнику знати внутрішню структуру ролей та API, не маючи прямого доступу до серверної бази даних. Це дало можливість:

  • Перевірити граничні стани дозволів (canAccess)
  • Симулювати різні відповіді API (success/error/HTML)
  • Тестувати поведінку при worker_name = null
  • Перевірити fallback AsyncStorage при недоступному API
// Grey-box: знаємо логіку, тестуємо через UI it('massage НЕ бачить Elos-епіляція', () => { expect(canAccess('Elos-епіляція', 'massage')).toBe(false); }); it('fallback: пустий worker → «Не вказано»', () => { const raw = { ...record, worker_name: null }; expect(mapRecord(raw).master).toBe('Не вказано'); });
✦ 08 · ВПРОВАДЖЕННЯ

Встановлення .apk на Android

На поточному етапі застосунок розповсюджується як .apk-файл для пристроїв на базі ОС Android. Публікація в App Store (iOS) не здійснена у зв'язку з відсутністю фінансування облікового запису розробника Apple Developer Program ($99/рік).

1️⃣

Збірка .apk через EAS Build

Використовується Expo Application Services (EAS Build). Конфігурація в eas.json. Команда: eas build --platform android

2️⃣

Передача файлу замовнику

Файл .apk передається через Google Drive, Telegram або пряме посилання. Підходить для внутрішнього корпоративного використання.

3️⃣

Дозвіл «Невідомі джерела»

На пристрої перейти: Налаштування → Безпека → Встановлення з невідомих джерел → увімкнути для браузера/файлового менеджера.

4️⃣

Встановлення .apk

Відкрити завантажений файл → натиснути «Встановити» → дочекатись завершення.

5️⃣

Перший запуск і авторизація

Запустити Beauty Room Workspace → ввести логін/пароль, наданий адміністратором системи.

📋
Системні вимоги
  • ОС: Android 10.0 (API 29) або новіше
  • RAM: мінімум 2 GB
  • Сховище: ~50 MB вільного простору
  • Інтернет: обов'язковий (Wi-Fi або мобільний)
  • Дозволи: Сповіщення, Галерея (для фото)
🍎
Щодо iOS
Застосунок розроблено на Expo / React Native, що технічно дозволяє збірку під iOS. Однак публікація в App Store вимагає членства в Apple Developer Program ($99/рік), якого наразі немає. При появі відповідного фінансування — застосунок може бути опублікований без значних змін у коді.

🔄 Оновлення: При виході нової версії — надається новий .apk файл. Встановлення поверх існуючого зберігає дані сесії в AsyncStorage. У майбутньому можлива автоматизація оновлень через Expo Updates (OTA).
